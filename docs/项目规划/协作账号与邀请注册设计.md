# 协作账号与邀请注册设计

## 1. 目标与边界

本设计为项目组远程协作补足正式的管理端认证能力。管理员可初始化首个账号、创建一次性教师邀请，受邀队友完成注册后可进入管理端处理模拟名单、证书生成、存证和 PDF 验真联调。

- 不开放管理员自助注册，也不允许注册时自选角色。
- 管理后台优先使用独立的 `admin.verify.lotusrain.net`；待 DNS 和 HTTPS 验收后迁移至该子域。
- 在子域 DNS 尚未就绪期间，`verify.lotusrain.net` 可临时提供同域受限协作入口，仅放行登录、受邀注册、指定管理页面、`/api/auth/*` 和 `/api/admin/*`；真实授权仍由后端 JWT 与 RBAC 执行。
- 学生正式登录、密码重置、邮件发送和真实个人信息导入不属于本轮范围。

## 2. 账号与权限

| 角色 | 创建方式 | 权限 |
| --- | --- | --- |
| `ADMIN` | 服务器交互式初始化或已有管理员创建 | 账号、邀请和全部证书业务操作 |
| `TEACHER` | 管理员创建的一次性邀请注册链接 | 模板、模拟学生导入、批次、签发、存证、撤销和补发 |
| `AUDITOR` | 预留受控创建，当前管理员邀请接口不创建该角色 | 只读查看回执和审计日志 |
| `STUDENT` | 本轮不创建 | 后续必须绑定既有学生记录，不能用任意学号替代身份 |

所有 `/api/admin/**` 请求都必须在 FastAPI 后端执行令牌与角色检查；Vue 路由和 Caddy 只作为界面与网络层补充，不能代替接口鉴权。

## 3. 数据与令牌

- `users`：保存用户名、显示名、密码哈希、角色、启用状态和创建时间。
- `invitations`：仅保存邀请码哈希、目标角色、创建者、过期时间和使用时间。邀请码明文只在创建响应中返回一次，不写入数据库或日志。
- `auth_sessions`：保存 JWT 的 `jti`、用户、有效期和撤销时间，实现登出与禁用账号后立即失效。
- 密码使用 Argon2 哈希；JWT 使用服务器 `JWT_SECRET` 签名，包含 `sub`、`role`、`jti`、签发时间和过期时间。

## 4. 接口

| 方法 | 路径 | 访问边界 |
| --- | --- | --- |
| `POST` | `/api/auth/login` | 已启用账号 |
| `POST` | `/api/auth/logout` | 已登录账号 |
| `GET` | `/api/auth/me` | 已登录账号 |
| `POST` | `/api/auth/register/invitation` | 有效、未使用邀请码 |
| `GET` | `/api/admin/users` | `ADMIN` |
| `PATCH` | `/api/admin/users/{id}/status` | `ADMIN` |
| `POST` | `/api/admin/invitations` | `ADMIN` |

## 5. 发布门禁与回滚

1. 从最新 `dev` 的隔离工作树构建，先备份 MySQL、证书输出目录和 Caddy 配置。
2. 执行只新增表的升级脚本，并在服务器交互式创建首个管理员；密码不出现在命令历史、仓库或部署环境文件。
3. 验证未登录 `401`、越权 `403`、邀请过期/重复使用失败、禁用账号和登出后令牌失效。
4. 验证受邀教师可完成模拟名单导入、证书签发、存证和公开 PDF 验真。
5. 当前同域回退持续拒绝 `/docs`、`/openapi.json`、健康接口和其他非白名单 API。2026-07-23 经项目负责人批准，临时放行 `/student/*` 与 `/api/student/certificates` 及其子路径，仅用于模拟 `student_no` 查询、下载和二维码演示；完成演示后应恢复拦截或先完成学生账号绑定与 JWT。待 DNS、Caddy HTTPS 和浏览器回归通过后，将管理入口迁移至 `admin.verify.lotusrain.net`；迁移后公共验真域可恢复为仅验真白名单。

认证表为新增表，应用回滚不需要删除它们。若迁移或验收失败，先恢复 Caddy 为仅公共验真，再恢复部署前数据库备份；不删除已签发证书、回执或审计记录。

## 6. 决策记录

| 决策 | 备选方案 | 依据 |
| --- | --- | --- |
| 使用邀请注册 | 任意用户公开注册 | 项目组协作需要可撤销访问，管理员权限不能公开授予 |
| 应用内认证 + 管理子域优先/同域受限回退 | 仅 Caddy Basic Auth | 后端接口必须独立鉴权，不能依赖网关或前端；DNS 未就绪不能阻塞受控团队联调 |
| 不做学生注册 | 立即做全量学生账号 | 当前学生端仍是演示身份模式，优先保证证书主闭环和团队协作 |
