# 目录说明

本文档是项目目录和文档资产的当前索引。新文件优先放入已有职责目录，避免在项目根目录堆放临时材料。

## 项目结构

```text
暑期实训/
├─ AGENTS.md                       # Codex 协作规则、项目边界和验收口径
├─ README.md                       # 项目总入口
├─ backend/                        # FastAPI 后端、模型、服务、脚本和测试
├─ frontend/                       # Vue 3 管理员端
├─ docs/
│  ├─ 原始材料/                    # 课程发布材料原始副本，只读保存
│  ├─ 提交模板/                    # 学校提供的空白 Word 模板
│  ├─ 提交内容/                    # 基于模板形成的当前填写稿
│  ├─ 项目规划/                    # 范围收束、进度安排和目录索引
│  ├─ 协作管理/                    # 接口、数据库、分工、联调和演示记录
│  └─ 过程文档/                    # 文本提取脚本及中间分析材料
├─ scripts/                        # 仓库辅助脚本
├─ outputs/                        # 本地生成的证书或演示产物，Git 忽略
└─ logs/                           # 本地运行日志，Git 忽略
```

## 文档职责

| 目录或文件 | 用途 | 维护规则 |
| --- | --- | --- |
| `AGENTS.md` | 项目级协作规则和不可越过的边界 | 只有项目规则或验收口径变化时更新 |
| `README.md` | 新成员的项目入口 | 保持结构、运行方式和当前状态准确 |
| `docs/项目规划/开发目标收束.md` | 功能范围、优先级、风险和里程碑 | 重要功能或范围变化后同步 |
| `docs/协作管理/接口规范.md` | 前后端接口和字段契约 | 代码契约变化时同步 |
| `docs/协作管理/数据库设计.md` | 数据模型和一致性约束 | 表结构或关键字段变化时同步 |
| `docs/协作管理/每日联调记录.md` | 按日期记录集成结果和阻塞项 | 不覆盖历史记录，新增日期章节 |
| `docs/提交内容/` | 当前可继续修改的课程文档填写稿 | 文件名保持稳定，不保留 `(1)`、`(2)` 等下载副本后缀 |

## 归档约定

1. `原始材料/` 和 `提交模板/` 不直接编辑，需要修改时复制到 `提交内容/`。
2. 阶段性分工和交付记录必须在标题或文件名中保留绝对日期，避免被误认为当前状态。
3. 页面截图放在对应文档旁的 `screenshots/`，不要散落在项目根目录。
4. 运行日志、构建产物、缓存和生成证书留在 `logs/`、`outputs/` 或各模块的忽略目录，不提交 Git。
5. 新建顶级目录前先确认现有目录无法承载该职责，避免重复出现 `temp`、`new`、`最终版` 等不可追踪目录。
