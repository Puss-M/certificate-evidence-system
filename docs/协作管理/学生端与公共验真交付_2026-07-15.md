# 学生端与公共验真端交付记录（2026-07-15）

## 负责范围

本次提交覆盖学生端、公共验真端与验真测试支撑，服务验收演示中的以下步骤：

1. 学生查看本人证书、下载 PDF、展示二维码并分享验真链接。
2. 第三方通过证书编号或二维码链接验真。
3. 第三方上传 PDF 复验，展示哈希一致或哈希不一致。
4. 撤销证书、未知证书等异常场景能返回明确结果。

## 后端接口

新增学生端接口：

- `GET /api/student/certificates?student_no=S20260001`
- `GET /api/student/certificates/{certificate_no}?student_no=S20260001`
- `GET /api/student/certificates/{certificate_no}/download?student_no=S20260001`
- `GET /api/student/certificates/{certificate_no}/qrcode?student_no=S20260001`

当前阶段没有完整学生登录系统，因此用 `student_no` 作为演示版身份标识。后续接入学生账号后，应从登录 token 中读取学生身份，避免前端传参。

该方式只适用于模拟数据演示：知道他人学号的访问者仍可发起查询，因此不能表述为已经完成正式账号鉴权或生产级身份隔离。

复用已有公共验真接口：

- `GET /api/verification/{certificate_no}`
- `POST /api/verification/{certificate_no}/file`

## 前端页面

新增公开路由：

- `/student`：学生个人中心
- `/student/certificates`：我的证书
- `/student/certificates/:certificateNo`：证书详情、PDF 下载、二维码展示
- `/public/verify`：公共验真入口
- `/public/verify/:certificateNo`：二维码/链接验真入口

公共验真结果页展示字段包括：证书编号、学生姓名、项目名称、当前状态、哈希是否一致、回执是否存在、回执编号、系统存证哈希、上传文件哈希、撤销原因和撤销时间。

## 验收演示建议

1. 使用管理员端生成并存证一张证书。
2. 打开 `/student?student_no=S20260001`，进入“我的证书”。
3. 打开证书详情，展示 PDF 下载、二维码和存证回执。
4. 打开 `/public/verify/{certificate_no}`，展示验真通过。
5. 上传原 PDF，展示哈希一致。
6. 修改 PDF 后再次上传，展示哈希不一致。
7. 管理员撤销该证书后再次访问验真页，展示证书已撤销。

## 2026-07-16 测试链适配

队伍新增 `chain/` 本地测试链和批次 Merkle Root 上链能力后，本模块完成以下适配：

- 公共验真页在编号验真或上传 PDF 验真后，会额外查询 `/api/verification/{certificate_no}/merkle-proof`。
- 验真结果页分开展示证书当前状态、PDF 哈希校验、Merkle Proof 和测试链交易回执，避免把“内容完整性通过”和“证书当前有效”混为一谈。
- 当批次尚未生成 Merkle Root 或测试链未配置时，页面显示“暂无 Merkle Root / 未写入测试链”，但不影响编号验真和 PDF 哈希复验主流程。
- 学生证书详情页新增 Root 编号展示，方便学生确认自己的证书是否已纳入批次 Root。
- 后端 `MerkleProofResult` 增加 `leaf_count`、`previous_root_hash`、`current_root_hash`、`tx_hash` 字段，便于公共验真端展示链式回执和测试链交易哈希。

二维码默认应配置为前端 `/public/verify/{certificate_no}` 页面。若此前二维码指向 `/api/verification/{certificate_no}`，需更新 `PUBLIC_VERIFY_BASE_URL` 后重新生成证书；旧二维码不会自动改写。
