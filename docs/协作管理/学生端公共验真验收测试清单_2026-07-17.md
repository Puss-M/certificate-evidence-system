## 公共验真状态覆盖

Mock 演示模式下建议使用以下编号：

| 状态 | 操作数据 | 验收方式 |
| --- | --- | --- |
| `PASS` | `CERT-20260713-0001` | 编号验真通过，显示哈希一致、回执存在 |
| `HASH_MISMATCH` | 任一存在证书 + 文件名包含 `tamper` 或 `篡改` 的 PDF | 上传 PDF 后显示哈希不一致 |
| `REVOKED` | `CERT-20260710-0003` | 显示证书已撤销 |
| `REISSUED` | `CERT-20260712-0011` | 显示旧证书已补发 |
| `NOT_FOUND` | `CERT-NOT-EXIST` | 显示未查询到该证书编号 |
| `NO_RECEIPT` | `CERT-20260713-0002` | 显示证书存在但未完成存证或回执不存在 |

真实后端模式下，以上状态应通过数据库造数或管理员端操作触发；前端页面读取后端 `verify_result` / `verify_message` 字段展示，不应在页面硬编码真实结果。

## PDF 上传测试

| 用例 | 操作 | 期望结果 |
| --- | --- | --- |
| 原始 PDF | 输入证书编号，上传未修改的证书 PDF | `PASS`，上传文件哈希与存证哈希一致 |
| 篡改 PDF | 修改 PDF 后上传，或 Mock 模式上传文件名含 `tamper`/`篡改` 的文件 | `HASH_MISMATCH`，上传文件哈希与存证哈希不一致 |

## 测试链 / Merkle Proof 展示

| 场景 | 期望结果 |
| --- | --- |
| 批次已生成 Merkle Root | 公共验真页展示 Root 编号、Merkle Root、Proof 验证结果、Root 链哈希 |

| 未配置测试链或未生成 Root | 显示暂无 Merkle Root / 未写入测试链，不影响编号验真和 PDF 验真 |
