# 演示验收脚本

本文档用于固定答辩演示流程。所有功能开发和测试都应围绕这条脚本推进，避免页面很多但主流程跑不通。

## 1. 演示目标

向老师展示系统可以完成：

1. 管理员批量导入学生名单。
2. 系统生成证书编号、PDF 和二维码。
3. 系统计算证书 SHA-256 并完成存证。
4. 学生下载和分享证书。
5. 第三方扫码验真并看到有效结果。
6. 上传篡改 PDF 后看到哈希不一致。
7. 管理员撤销证书后，再次验真显示已撤销。

## 3. 标准演示流程

### 步骤 1：管理员登录

1. 打开线上登录页 `https://verify.lotusrain.net/login`。
2. 使用已初始化的管理员账号登录；如需队友操作，由管理员在“协作账号”创建一次性教师邀请链接。
3. 进入后台首页；受邀教师完成注册后使用自己的账号参与后续步骤。

预期结果：

- 登录成功。
- 后台首页显示证书数量、批次数量、存证数量和验真数量。
- 不共享管理员密码；学生端不作为公网身份入口。

### 步骤 2：创建证书模板

1. 进入证书模板页面。
2. 创建“暑期实训结业证书”模板。
3. 填写颁发机构、项目名称、证书正文和日期。

预期结果：

- 模板保存成功。
- 模板列表能看到新模板。

### 步骤 3：导入学生名单

1. 进入学生或批次页面。
2. 上传 `students_demo.xlsx`。
3. 创建“2026 暑期实训第一批”证书批次。

预期结果：

- 导入 5-10 名模拟学生。
- 系统生成批次记录。
- 批次状态为 `IMPORTED` 或类似状态。

### 步骤 4：批量生成证书

1. 点击批量生成证书。
2. 系统生成证书编号、PDF 和二维码。
3. 查看证书列表。

预期结果：

- 每名学生都有唯一 `certificate_no`。
- 每张证书都有 `pdf_path`、`verify_url`、`certificate_hash`。
- 可预览或下载 PDF。

### 步骤 5：执行存证

1. 管理员点击批量存证。
2. 系统将证书哈希写入本地哈希链或测试链。
3. 查看存证回执。

预期结果：

- 每张证书都有 `receipt_id`。
- 页面显示存证时间、哈希、回执编号。
- 如 FISCO 未接入，明确显示“本地哈希链存证”。

### 步骤 6：证书下载与二维码交付

1. 在线上后台确认生成的 PDF 和二维码产物。
2. 临时演示入口 `/student` 使用模拟 `student_no` 查询“我的证书”，再下载 PDF 或展示二维码。
3. 将二维码用于公共验真；不把学号查询写成正式登录，也不使用真实学生资料。

预期结果：

- 证书 PDF 与二维码产物存在，二维码可打开公共验真页。
- 临时学生页可展示演示数据的查询、下载和二维码；完成演示后应恢复拦截，或在学生身份绑定完成后替换为正式登录。

### 步骤 7：扫码验真成功

1. 使用二维码或证书编号进入公共验真页。
2. 查询证书状态和哈希。

预期结果：

- 显示“证书有效”。
- 显示“哈希一致”。
- 显示“存证回执存在”。

### 步骤 8：上传篡改 PDF 验真失败

1. 准备一份内容被修改过的 PDF。
2. 在公共验真页上传该 PDF。
3. 系统重新计算 SHA-256。

预期结果：

- 显示“哈希不一致”。
- 显示上传文件哈希和系统存证哈希不同。
- 不改变系统中原始证书记录。

### 步骤 9：撤销证书

1. 管理员进入证书列表。
2. 选择刚才验真的证书。
3. 填写撤销原因并提交。

预期结果：

- 证书状态变为 `REVOKED`。
- 操作日志记录撤销人、撤销时间和原因。

### 步骤 10：撤销后再次验真

1. 再次扫描同一张证书二维码。
2. 查看公共验真结果。

预期结果：

- 显示“证书已撤销”。
- 即使哈希一致，也不再显示证书有效。
- 页面说明当前有效性依赖系统状态和撤销记录。

## 4. 异常测试清单

| 测试项 | 操作 | 预期结果 |
| --- | --- | --- |
| 不存在证书 | 输入不存在的证书编号 | 显示未查询到证书 |
| 篡改 PDF | 上传修改后的 PDF | 显示哈希不一致 |
| 已撤销证书 | 扫描撤销后的二维码 | 显示证书已撤销 |
| 未存证证书 | 查询只有 PDF 但无回执的证书 | 显示未完成存证 |
| 重复撤销 | 对已撤销证书再次撤销 | 显示状态冲突或提示已撤销 |
| Excel 格式错误 | 上传缺少字段的 Excel | 显示导入失败原因 |

## 5. 答辩说明话术

### 5.1 为什么使用本地哈希链

本项目两周内优先完成证书生命周期闭环。系统当前使用 SHA-256 和本地哈希链模拟可信存证，能够证明证书 PDF 在存证后是否被篡改。接口层已预留 FISCO BCOS 联盟链接入能力，后续可将本地回执替换为链上交易回执。

### 5.2 哈希存证能证明什么

哈希存证证明的是：某个证书文件版本在存证后没有被修改。如果有人修改 PDF，重新计算出的 SHA-256 会和系统保存的哈希不同，因此验真失败。

### 5.3 哈希存证不能证明什么

哈希不能单独证明证书内容在现实中一定真实，也不能证明证书现在仍然有效。当前有效性还需要结合学校签发权限、系统状态、撤销记录和审计日志判断。

## 7. FISCO 不可用时的固定演示方案

如果答辩现场 FISCO BCOS 节点、合约或网络不可用，直接使用以下本地哈希链固定演示方案，不临场排查链环境。

### 7.1 固定演示数据

| 数据 | 要求 |
| --- | --- |
| 原始 PDF | 至少准备 1 张可验真通过的证书 PDF |
| 篡改 PDF | 至少准备 1 张内容被修改的 PDF |
| 本地回执 | 至少准备 1 条本地哈希链回执记录 |
| 截图 | 提前保存“有效、哈希不一致、已撤销”三类结果截图 |

### 7.2 演示切换话术

> 当前演示使用本地哈希链作为教学版可信存证环境。系统已通过统一存证接口隔离底层实现，因此可以在不改变发证、验真和撤销业务流程的情况下，将本地回执替换为 FISCO BCOS 的链上交易回执。

### 7.3 不允许现场冒险

1. 不在答辩现场临时重启链节点。
2. 不在答辩现场临时重新部署合约。
3. 不因为链环境失败而跳过扫码验真、篡改 PDF 验真和撤销验真。
4. 如果 FISCO 演示不稳定，只展示本地哈希链主流程和 FISCO 预留设计。
