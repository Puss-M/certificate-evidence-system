## 三、页面问题清单

以下仅列出尚未完全解决的问题。

| 页面/模块 | 类型 | 问题 | 优先级 | 建议处理方 |
| --- | --- | --- | --- | --- |
| 批次统计 | 字段 | `generated`、`evidenced` 与 `student_count` 的统计口径需要由后端保持一致，避免进度超过 100% | 高 | 后端 |
| 批次列表 | 字段/性能 | 批次列表未直接返回 `root_id` 或 `has_merkle_root`，前端需要逐批次查询 Root | 中 | 后端接口优化 |
| 证书列表 | 字段 | 已存证证书需要稳定返回 `receipt_id`；缺失时只能按 `certificate_no` 查询回执 | 高 | 后端 |
| 批量签发 | 交互 | 尚未提前过滤与项目不匹配的模板和批次，主要依赖提交后的后端校验 | 中 | 前后端 |
| 批量签发 | 交互 | 部分签发失败后暂不支持只重试失败学生 | 低 | 前端优化 |
| 学生导入 | 字段/验收 | 仍需验证后端在部分失败时完整返回成功数、失败数、失败行号和原因 | 高 | 后端及联调 |
| 全局表单 | 交互 | 项目、模板、批次等部分新增、编辑、删除操作尚未全部统一提交中状态 | 中 | 前端 |
| 角色权限 | 验收 | ADMIN、TEACHER、AUDITOR 仍需使用真实账号完成菜单、路由和操作权限全流程回归 | 高 | 前后端联调 |
| 生产构建 | 性能 | 主 JavaScript chunk 约 1.1 MB，存在超过 500 KB 的非阻塞警告 | 低 | 前端后续优化 |

## 四、路由与页面说明

本地启动：

```bash
npm run dev
```

默认访问地址：`http://localhost:5173`

| 页面 | 路由 | 复现说明 |
| --- | --- | --- |
| 登录页 | `/login` | 清除登录状态后访问，使用管理员账号登录 |
| 后台首页 | `/dashboard` | ADMIN 登录后查看统计数据和最近记录 |
| 实训项目 | `/projects` | 查看项目列表及新增、编辑、删除操作 |
| 学生管理 | `/students` | 查看学生列表，点击“Excel导入”验证导入结果 |
| 证书模板 | `/templates` | 查看模板字段、签发机构和预览入口 |
| 证书批次 | `/batches` | 查看批次进度；点击“详情”查看 Root，未生成时可点击“生成Root” |
| 证书管理 | `/certificates` | 查看证书冻结字段；已存证证书点击“查看回执” |
| 存证回执 | `/chain` | 查询证书编号或回执编号，点击“详情”查看哈希链信息 |
| 操作日志 | `/audit` | 查看关键操作、操作者和操作时间 |
| 公共验真 | `/public/verify/:certificateNo` | 将 `:certificateNo` 替换为真实证书编号，查看证书及批次级 Merkle Proof |

## 五、验收结论

管理员端核心业务页面已经能够展示批次、证书、Merkle Root 和存证回执信息，关键重复提交问题已处理，生产构建能够通过。当前截图包对批次级存证和证书模板覆盖较完整，但证书列表冻结字段、证书详情新旧关联和学生导入结果的截图仍需补充后再作为完整交付包。
