## 3. 权限与状态码

管理接口统一要求 `Authorization: Bearer <token>`。

| 场景 | HTTP 状态码 | 前端处理建议 |
| --- | --- | --- |
| 未登录、token 缺失或失效 | `401` | 清理本地登录状态并返回登录页 |
| `AUDITOR` 发起新增、修改、删除等写操作 | `403` | 提示无操作权限，不重复提交 |
| 请求字段或日期格式错误 | `422` | 保留表单并标出错误字段 |
| 记录不存在 | `404` | 提示数据已不存在并刷新列表 |
| 重复撤销、重复补发、删除已被引用数据等状态冲突 | `409` | 展示后端原因并刷新当前状态 |
| 业务前置条件不满足，例如缺少证书哈希 | `400` | 提示先完成前置步骤 |
| 服务内部异常 | `500` | 保留现场，查看后端日志，不展示堆栈或配置 |

实测结果：未登录读取管理接口返回 `401`；审计员读取返回 `200`、写入返回 `403`；管理员提交结束日期早于开始日期返回 `422`。主要管理端只读接口均返回 `200`。公共验真对不存在编号沿用当前兼容契约：HTTP 返回 `200`，业务字段返回 `NOT_FOUND`。

## 4. 持久化检查

1. 执行 `python -m scripts.create_tables`，结果为建表检查通过。
2. 执行 `python -m scripts.upgrade_certificate_schema`，结果为结构升级通过。
3. MySQL 重启前后分别读取表名和各表行数，只比较 SHA-256 摘要，不导出业务内容。
4. 重启前后均检测到 10 张表，行数摘要一致，说明现有数据已落盘并在重启后保留。
5. 重启后 `/api/health` 与 `/api/health/db` 均返回 `UP`。

迁移脚本采用幂等方式补表、补字段和索引，不删除历史数据。若唯一索引遇到历史重复值，脚本会中止并要求人工处理，不会自动删记录。

## 7. 安全要求

- `.env`、数据库目录、日志、生成证书、二维码和密钥文件不得提交。
- 跨电脑联调地址使用 `http://<后端电脑局域网IP>:8000` 等占位说明，实际地址只在队内临时传递。
- 日常单机开发让 FastAPI 监听 `127.0.0.1`；仅跨电脑联调期间临时监听 `0.0.0.0`，结束后立即恢复。
- 测试链只允许项目约定的本地链 ID，不连接主网，不使用真实钱包、真实身份信息或真实证书数据。
