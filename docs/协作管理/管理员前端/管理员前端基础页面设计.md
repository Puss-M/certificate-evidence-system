# Vue 3 管理员端基础页面设计

本项目在现有 `admin-web` 中继续开发，不重新创建工程。当前阶段使用 Vue 3、TypeScript、Vue Router、Pinia、Axios 和 Element Plus，只实现基础管理页面与 Mock 数据。

## 页面路由

| 路由 | 页面 | 主要内容 |
| --- | --- | --- |
| `/login` | `LoginView.vue` | 用户名、密码、登录 |
| `/dashboard` | `DashboardView.vue` | 项目、学生、证书和存证数量概览 |
| `/students` | `StudentsView.vue` | 学生查询、新增、编辑、删除、Excel 导入 |
| `/templates` | `TemplatesView.vue` | 证书模板查询、新增、编辑、删除、预览 |
| `/batches` | `BatchesView.vue` | 证书批次查询、新增、编辑、删除、详情 |
| `/certificates` | `CertificatesView.vue` | 证书查询、详情和基础操作 |

现有 `/projects`、`/chain`、`/audit` 页面继续保留，不影响本阶段基础页面。

## 组件拆分

- `layout/AdminLayout.vue`：后台整体布局、侧边菜单和顶部用户区域。
- `components/PageHeader.vue`：页面标题、说明和页面级操作按钮。
- `components/StatusTag.vue`：证书、批次和存证状态标签。
- `views/LoginView.vue`：登录表单。
- `views/DashboardView.vue`：首页统计卡片和最近记录。
- `views/StudentsView.vue`：学生列表、编辑弹窗和 Excel 导入弹窗。
- `views/TemplatesView.vue`：模板列表、编辑弹窗和基础预览。
- `views/BatchesView.vue`：批次列表、编辑弹窗和详情抽屉。
- `views/CertificatesView.vue`：证书列表和详情抽屉。
- `api/*.ts`：页面只调用 API 函数，不直接写 URL。
- `api/mock.ts`：统一保存本地 Mock 数据和分页逻辑。
- `types/index.ts`：统一保存 TypeScript 数据类型。

## 证书列表冻结字段

证书列表直接使用以下后端字段，不转换为 camelCase：

- `certificate_no`
- `student_name`
- `batch_id`
- `certificate_hash`
- `receipt_id`
- `status`

## FastAPI 接口清单

### 登录与首页

- `POST /api/auth/login`：管理员登录。
- `GET /api/admin/dashboard/statistics`：后台首页统计。

### 学生

- `GET /api/admin/students`：分页查询学生。
- `POST /api/admin/students`：新增学生。
- `PUT /api/admin/students/{student_id}`：编辑学生。
- `DELETE /api/admin/students/{student_id}`：删除学生。
- `POST /api/admin/students/import`：导入 Excel 名单。

### 证书模板

- `GET /api/admin/templates`：分页查询模板。
- `POST /api/admin/templates`：新增模板。
- `PUT /api/admin/templates/{template_id}`：编辑模板。
- `DELETE /api/admin/templates/{template_id}`：删除模板。

### 证书批次

- `GET /api/admin/certificate-batches`：分页查询批次。
- `POST /api/admin/certificate-batches`：新增批次。
- `PUT /api/admin/certificate-batches/{batch_id}`：编辑批次。
- `DELETE /api/admin/certificate-batches/{batch_id}`：删除批次。

### 证书

- `GET /api/admin/certificates`：分页查询证书。
- `GET /api/admin/certificates/{certificate_id}`：查询证书详情，后端后续实现。

统一响应格式：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {}
}
```

分页 `data`：

```json
{
  "records": [],
  "total": 0,
  "current": 1,
  "size": 10
}
```
