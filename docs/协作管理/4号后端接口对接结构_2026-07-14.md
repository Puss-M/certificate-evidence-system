# 4号后端接口对接结构（2026-07-14）

本文档说明 4 号后端当前提供的接口、数据流和各成员对接点。文档只写项目相对路径、接口路径和演示账号，不写本机绝对路径、数据库真实密码、密钥、私钥或真实学生信息。

## 1. 当前系统结构

```text
frontend/
  src/api/                 Vue 3 管理员端接口封装
  src/views/               管理员端页面
  vite.config.ts           本地代理 /api -> http://127.0.0.1:8000

backend/
  app/main.py              FastAPI 应用入口、CORS、异常处理
  app/api/router.py        统一注册 /api 下的路由
  app/api/routes/auth.py   管理员演示登录
  app/api/routes/admin.py  管理员端对接接口
  app/api/routes/verification.py  公共验真接口
  app/services/certificate_service.py  证书生成与本地哈希链回执
  app/services/verification_service.py  编号验真与 PDF 哈希复验
  app/models/              SQLAlchemy 表模型
  scripts/create_tables.py 建表
  scripts/upgrade_certificate_schema.py 旧表安全补列
```

## 2. 核心数据流

```text
管理员前端
  -> /api/auth/login
  -> /api/admin/students / templates / batches
  -> /api/admin/batches/{batch_id}/generate

后端签发接口
  -> certificate_service.generate_certificate()
  -> 生成证书编号、PDF、二维码
  -> 计算 PDF SHA-256
  -> 写 certificates
  -> 写 evidence_receipts，本地哈希链 block_height 递增

公共验真端
  -> /api/verification/{certificate_no}
  -> /api/verification/{certificate_no}/file
  -> verification_service 校验证书状态、回执和上传 PDF 哈希
```

## 3. 与 5 号证书和链路模块的对接

5 号已经接入的主线在 `backend/app/services/certificate_service.py`：

- `generate_certificate()`：单张证书生成、PDF/二维码生成、证书哈希计算、写入证书表和回执表。
- `generate_certificate_batch()`：批量调用单张生成逻辑。
- `_create_evidence_receipt()`：创建本地哈希链回执，维护 `previous_hash`、`current_block_hash`、`block_height`。

本轮已按 5 号字段核对补齐：

- `certificates.project_name`
- `certificates.issue_time`
- `certificates.previous_certificate_no`
- `certificates.updated_at`
- `revocation_records.action_type`
- `revocation_records.new_certificate_no`

补发规则：

- 新证书写入 `previous_certificate_no = 旧证书编号`。
- 旧证书状态更新为 `REISSUED`。
- 公共验真遇到 `REISSUED` 不返回 `PASS`。
- 旧证书不物理删除，保留审计和追溯。

## 4. 与 2 号管理员前端的对接

前端设置：

- 本地开发服务端口：`5173`
- Vite 已配置 `/api` 代理到后端 `8000`
- 关闭 mock 后，前端请求会进入 FastAPI 后端

前端当前请求路径均已补齐后端接口：

| 前端模块 | 后端接口 |
| --- | --- |
| 登录 | `POST /api/auth/login` |
| 仪表盘 | `GET /api/admin/dashboard/statistics` |
| 项目管理 | `GET/POST/PUT/DELETE /api/admin/projects` |
| 学生管理 | `GET/POST/PUT/DELETE /api/admin/students` |
| Excel 导入入口 | `POST /api/admin/students/import` |
| 模板管理 | `GET/POST/PUT/DELETE /api/admin/templates` |
| 批次管理 | `GET/POST/PUT/DELETE /api/admin/batches` |
| 证书列表 | `GET /api/admin/certificates` |
| 批量签发 | `POST /api/admin/batches/{batch_id}/generate` |
| 单张存证 | `POST /api/admin/certificates/{certificate_id}/evidence` |
| 批量存证 | `POST /api/admin/batches/{batch_id}/evidence` |
| 撤销 | `POST /api/admin/certificates/{certificate_id}/revoke` |
| 补发 | `POST /api/admin/certificates/{certificate_id}/reissue` |
| 回执列表 | `GET /api/admin/evidence/receipts` |
| 回执详情 | `GET /api/admin/evidence/receipts/{receipt_id}` |
| 链完整性检查 | `GET /api/admin/evidence/integrity` |
| 操作日志 | `GET /api/admin/audit-logs` |

说明：

- `projects` 等辅助接口当前是演示兼容层，保证前端页面能联调。
- `certificates`、`evidence_receipts`、公共验真主链路已经接数据库和证书服务。
- 前端响应拦截器已兼容后端统一返回 `code = 0`。

## 5. 与 3 号公共验真端的对接

公共验真接口：

| 场景 | 接口 | 说明 |
| --- | --- | --- |
| 编号/扫码验真 | `GET /api/verification/{certificate_no}` | 校验证书是否存在、状态是否有效、是否有回执 |
| 上传 PDF 复验 | `POST /api/verification/{certificate_no}/file` | 计算上传文件 SHA-256，与数据库 `certificate_hash` 比对 |

验真结果状态：

- `PASS`
- `REVOKED`
- `REISSUED`
- `HASH_MISMATCH`
- `NOT_FOUND`
- `NO_RECEIPT`
- `SYSTEM_ERROR`

## 6. 与 1 号集成部署的对接

后端本地启动：

```powershell
cd backend
.\.venv\Scripts\python.exe -m uvicorn app.main:app --reload
```

前端依赖和构建：

```powershell
cd frontend
npm install
npm run build
```

数据库建表：

```powershell
cd backend
.\.venv\Scripts\python.exe -m scripts.create_tables
```

如果旧数据库已经建过表，执行安全补列：

```powershell
cd backend
.\.venv\Scripts\python.exe -m scripts.upgrade_certificate_schema
```

该脚本只补缺失字段，不删表、不清数据。
