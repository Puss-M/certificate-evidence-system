
### 2.2 核心字段

以下字段已经冻结，所有页面 mock、后端接口、数据库、PDF/二维码/哈希模块都必须沿用。AI 只能补充展示字段或派生字段，不得重命名、替换、删除这些冻结核心字段：

| 字段 | 含义 | 今日要求 |
| --- | --- | --- |
| `certificate_id` | 系统内部证书主键 | 后端内部使用 |
| `certificate_no` | 对外展示证书编号 | 二维码、验真、页面展示统一使用 |
| `student_id` | 学生主键 | 关联学生记录 |
| `student_no` | 学号 | 使用模拟数据 |
| `student_name` | 学生姓名 | 使用模拟数据 |
| `batch_id` | 批量颁发批次 | 批量生成证书使用 |
| `template_id` | 证书模板主键 | PDF 生成使用 |
| `pdf_path` | PDF 文件位置 | 证书生成模块输出 |
| `certificate_hash` | PDF 的 SHA-256 | 对最终 PDF 计算 |
| `qr_code_path` | 二维码图片路径 | 可先用 mock 路径 |
| `verify_url` | 公共验真链接 | 二维码指向该链接 |
| `receipt_id` | 存证回执编号 | 本地哈希链输出 |
| `status` | 证书状态 | `VALID`、`REVOKED` 等 |
| `credential_type` | 凭证类型 | 当前默认 `CERTIFICATE` |
| `root_id` | Merkle Root 关联 | 今天可为空 |


## 3. 本机数据安全保护

今天每个人和自己的 AI 讨论时，必须遵守以下规则，防止泄露本机数据、账号信息和真实材料。

### 3.1 禁止发给 AI 的内容

| 类型 | 禁止内容 | 安全替代 |
| --- | --- | --- |
| 本机路径 | `C:\Users\真实用户名\...`、微信缓存路径、桌面私人目录 | 改成 `项目根目录/相对路径` 或 `<本机路径已省略>` |
| 密钥口令 | `.env`、数据库密码、JWT secret、API key、token、私钥、助记词 | 只给 `.env.example`，值写成 `CHANGE_ME` |
| 真实资料 | 真实学生姓名、学号、手机号、证书 PDF、身份证、聊天记录 | 使用模拟姓名、模拟学号、模拟机构 |
| 私人文件 | 微信临时文件、下载目录、个人截图、浏览器账号页 | 只截取项目页面，遮挡账号和路径 |
| 大段本机日志 | 含用户名、路径、token、数据库连接串的完整日志 | 删除敏感行后只保留报错摘要 |

### 3.2 和 AI 讨论时的安全写法

1. 只粘贴必要的代码片段，不上传整个本机目录。
2. 所有路径优先写相对路径，例如 `backend/app/main.py`、`frontend/src/views/CertificateList.vue`。
3. 所有账号密码示例统一写成 `admin / 123456` 或 `CHANGE_ME`，并标注“仅本地演示”。
4. 所有学生数据使用模拟数据，例如 `张三`、`20260001`、`示范学院`。
5. 截图前先检查是否包含浏览器账号、GitHub 私库信息、微信头像、Windows 用户名、完整本机路径。
6. 不让 AI 生成真实可用的密钥、私钥、助记词；如果需要配置，只生成 `.env.example`。
7. 如果不确定某段内容能不能发给 AI，先发给 1 号负责人检查。

### 3.3 提交前安全检查

每个人提交成果前必须确认：

- [ ] 没有 `.env`、数据库密码、JWT secret、API key、token、私钥、助记词。
- [ ] 没有真实学生信息、真实证书 PDF、真实身份证或手机号。
- [ ] 没有微信临时文件、个人下载文件、浏览器账号截图。
- [ ] 没有 `C:\Users\真实用户名\...` 这类完整本机隐私路径。
- [ ] 截图中没有账号、头像、私聊、通知栏和本机敏感路径。
- [ ] 交付说明只包含项目相关文件和相对路径。
